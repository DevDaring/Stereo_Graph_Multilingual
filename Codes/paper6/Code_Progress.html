Paper 6 - Code Progress (Robust_Rerun)

Leakage-free re-run of the Multilingual Stereotype Knowledge Graph (MS-SKG) study. Pipeline at Codes/paper6/Robust_Rerun/. Target: Elsevier Information Sciences special issue (Graph-based solutions for AI).

Report date: 2026-06-24 Stage: CPU outputs final (local, leakage gate passed); GPU run pending - pod SSH unreachable (provider-side banner-exchange timeout) Primary judge: LinkAPI gemini-2.5-pro (no auto-fallback)
Why a re-run. The previous Paper 6 results were withdrawn as inconsistent: the graph was built from the whole evaluation set (circular propagation, retrieval leakage), graph-RAG injected the query item's own anti-stereotype answer, and the "graph bridge" never used the graph. The old result files are moved to Codes/paper6/OLD_results_leaky_archived/ and must not be cited. All new outputs live under Robust_Rerun/results/ so there is no overlap.

1. What the re-run fixes

Defect (old)Fix (Robust_Rerun)
KG built from the entire eval set; propagation 0.767 circular; retrieval tested on graph-defining items. Concept-level train/val/test split; KG from TRAIN concepts only; retrieval on HELD-OUT TEST concepts; calibration on VAL.
Graph-RAG injected the query item's own anti-stereotype target (gold answer) - hence the over-correction. Retrieval draws counter-evidence only from TRAIN concepts and excludes the query's own surfaces and concept.
"Graph bridge" reused a target-language cut; residual units are not graph nodes. In-parameter bridge dropped as headline; disjointness re-run with 5 seeds + random-mask null; propagation replaced by held-out same_as link prediction vs a null.
No baselines for retrieval. Added: generic safety prompt, translate-to-English, flat dictionary, and a monolingual (no same_as) ablation.
Leakage cannot reappear - enforced, not just claimed. A hard gate (tests/test_leakage.py) runs inside run_all.py right after the KG is built and aborts the run on any violation. Verified on the real artifacts: every group node in the graph is a TRAIN concept (4,634 checked), the KG is built_from = train_concepts_only with 303 test concepts held out, and 0 of 4,572 retrieved facts equalled a query item's own answer on held-out TEST items.

Held-out propagation is an honest null. With half the same_as group links removed, PageRank places less mass on the true held-out counterparts than chance (mean bridge 0.435 vs null 0.546, lift 0.80). This confirms the old 0.767 was by construction: the explicit same_as edges carry the bridge; nothing latent recovers it. Reported as a strength (it cannot be attacked as circular). The graph's value is the inspectable artifact plus the weight-free retrieval intervention, not emergent propagation.

4. To run on the GPU VM

  1. pip install -r requirements.txt (global env, no venv)
  2. python install_flash_attention.py (pre-compiled wheel, Linux x86_64 + CUDA)
  3. python step_04_dry_run.py (confirm keys + ids)
  4. python run_all.py (runs 01-10; 05/08 GPU, 07/09 API)
Steps are resume-capable and write rows incrementally. Step 01 re-checks for duplicate/corrupted data on every invocation and hard-fails before any later step runs.
